# M0+M1 实施计划（工程初始化 + storage/compare P1）

> 依据：[00-overview](../design/00-overview.md)、[01-compare](../design/01-compare-enhancement.md)、[05-storage](../design/05-storage-enhancement.md)。
> 执行方式：subagent-driven development，每任务一个全新实现者 + 任务审查，最后整分支审查。
> 环境实测：Maven 3.8.8 / JDK 21.0.10 / server-facility 0.1.0-SNAPSHOT 已在本地 .m2 / Docker 29.2.0 可用。

## Global Constraints（全局约束，违反即缺陷）

1. Java 21、Spring Boot 3.5.10；版本一律由父 pom dependencyManagement 管理，子模块不写版本号。
2. 基础包 `cn.code91.toolbox.{module}`；配置前缀 `toolbox.{module}.*`；每模块唯一装配类 `Toolbox{Module}AutoConfiguration`，经 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 注册。
3. 对外 API 返回 `cn.code91.facility.result.Result<T, {Module}Error>`（facility 的 sealed Result）；`{Module}Error` 为 sealed interface + record 实现；SDK/底层异常一律在 adapter 的 ErrorMapper 内收敛为错误值，不得从公开 API 外抛。
4. 可选依赖（云 SDK 等）必须 `<optional>true</optional>`，装配处用 `@ConditionalOnClass(name = "全限定名字符串")` 探测（字符串形式，不 import SDK 类型）。
5. 每个默认 bean 挂 `@ConditionalOnMissingBean`（Seam，应用可覆盖）。
6. 复用 facility、禁止重造：HTTP 客户端/JSON/日志门面/脱敏/限流不得自建；storage 必须使用 `Filenames`/`MimeTyping`/`PathIo`；compare 的日期渲染用 `DateUtil`、i18n 用 `LocaleUtil`。
7. 不使用 Lombok；值对象用 record；注释与 Javadoc 用中文，只写代码本身表达不了的约束。
8. 质量门：jacoco BUNDLE INSTRUCTION/LINE ≥ 0.80、BRANCH ≥ 0.70；每模块含 ArchUnit 测试（三规则：包无环；autoconfigure 不被主包依赖；core/spi/guard 等门面包不依赖可选 SDK 类型）；`mvn verify` 全绿且测试输出无告警噪音。
9. 有产品代码的任务用 TDD：先写失败测试再实现，报告含 RED/GREEN 证据。
10. git 提交：中文一行主题；结尾带 `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`。

## Task 1: M0 工程初始化（父 POM 改造 + 模块骨架 + 构建验证）

目标：把空壳聚合工程改造为可构建、带质量门的多模块工程。设计依据 docs/design/00-overview.md §5。**本任务只改 pom 与构建配置，不写任何 Java 代码。**

改造项：

1. 父 `pom.xml`：
   - properties：删除 `maven.compiler.source/target`，改用 `maven.compiler.release=21`；保留 UTF-8 编码属性；集中声明版本属性（`spring-boot.version=3.5.10`、`server-facility.version=0.1.0-SNAPSHOT` 等）。
   - dependencyManagement：
     - import（pom/import scope）：`org.springframework.boot:spring-boot-dependencies:3.5.10`、`software.amazon.awssdk:bom:2.41.14`、`org.testcontainers:testcontainers-bom:1.20.4`；
     - pin：`cn.code91:server-facility:0.1.0-SNAPSHOT`、`com.aliyun.oss:aliyun-sdk-oss:3.18.1`、`com.tngtech.archunit:archunit-junit5:1.3.0`、`p6spy:p6spy:3.9.1`、`org.postgresql:postgresql:42.7.8`、`com.icegreen:greenmail-junit5:2.1.3`。
   - build：pluginManagement 并激活——
     - maven-compiler-plugin 3.13.0：release 走属性；`annotationProcessorPaths` 含 `org.springframework.boot:spring-boot-configuration-processor:${spring-boot.version}`（processorPath 需显式版本）；
     - maven-surefire-plugin 3.5.2：包含 `*Test` 与 `*IT`；**不要**自定义 argLine（jacoco 通过 `argLine` 属性注入 agent，若必须加参数要写成 `@{argLine} 其它参数`）；
     - jacoco-maven-plugin 0.8.12：prepare-agent + report + check@verify（BUNDLE：INSTRUCTION 0.80、LINE 0.80、BRANCH 0.70）；
     - maven-dependency-plugin 3.7.1：`analyze-only` 绑 verify，`failOnWarning=true`；`ignoredUnusedDeclaredDependencies` 预置测试脚手架类聚合 artifact（`org.springframework.boot:spring-boot-starter-test`、`com.tngtech.archunit:archunit-junit5`），后续模块出现新的聚合型误报可在此追加，但不许整体关闭 failOnWarning。
2. 修正 `llm-enhancement/pom.xml`：artifactId `llm-enhance` → `llm-enhancement`。
3. 五个模块 pom 统一为最小形态：仅 parent + artifactId（编码/release 由父统一，删除模块内重复 properties）；**本任务不给任何模块加依赖**（保证空模块下 analyze/jacoco 零告警通过）。
4. 验证：仓库根执行 `mvn -q verify` 全绿。若 jacoco/dependency 插件在空模块上告警失败，调整配置让空模块自然通过（例如 jacoco 无执行数据时跳过属正常行为），禁止用全局 skip。

验收标准：
- 仓库根 `mvn -q verify` exit 0；
- 五个模块 artifactId 与目录名一致；
- effective-pom 中编译 release=21、Spring Boot BOM 生效（spot check 任一模块）。

## Task 2: compare-enhancement P1（差异引擎 + 装配 + 测试）

必读设计（按序）：docs/design/00-overview.md §2–§4、docs/design/01-compare-enhancement.md 全文。P1 范围 = 01 文档 §10 P1 行，外加以下**控制器裁定**（本 brief 优先于设计文档的含糊处）：

- 裁定 A（i18n 提前到 P1）：`@CompareLabel` 的 `messageKey` 语义 P1 完整实现——`toolboxCompareMessageSource` bean + `i18n/toolbox-compare-messages*.properties`、经 facility `LocaleUtil` 解析，解析不到回退 `value`，`value` 为空回退字段名。避免 P2 变更注解语义。
- 裁定 B（集合收窄）：P1 集合仅实现 BY_INDEX（List/数组按下标逐位，元素为复杂对象时递归，长度不等的多余元素记 ADDED/REMOVED），Map 按 key 配对；`@CompareKey`、BY_KEY、AS_SET 全部留 P2，P1 不得出现相关代码。
- 裁定 C（渲染错误通道）：`ChangeRenderer.render(DiffResult)` 返回 `Result<String, CompareError>`（JsonRenderer 序列化失败转 `RenderError`，不抛异常）；`DiffResult.render(renderer)` 同签名委托。

交付物（基础包 `cn.code91.toolbox.compare`）：

- `annotation`：`@CompareLabel(value, messageKey)`、`@CompareIgnore`、`@CompareWith(Class<? extends ValueComparator<?>>)`（字段级，优先级最高）。
- `core`：`DiffEngine`（两个 diff 重载，见 01 §4.2）、`DiffResult`（record：`changes` + `identical()` + `render(...)`）、`FieldChange`（path/label/kind/oldValue/newValue/oldText/newText）、`ChangeKind`、`DiffOptions`（maxDepth 默认 8、nullAsEmpty 默认 false、includePaths/excludePaths）、`CompareError` sealed（TypeMismatch/DepthExceeded/CycleDetected/FieldAccessError/RenderError，record 实现，带 message 与路径定位）。
- `spi`：`ValueComparator<T>`、`ValueFormatter<T>`、`ChangeRenderer`、`CompareRegistryCustomizer`、`CompareHandlerRegistry`（内置 → `ObjectProvider` 收集 → customizer → `freeze()`；freeze 后 register 抛 `IllegalStateException`）。
- `engine`：`ReflectionDiffEngine` + `ClassMetaCache`（ConcurrentHashMap 按类缓存字段/accessor 元数据，record 与 JavaBean 双支持）。遍历规则严格按 01 §4.3：叶子类型表（BigDecimal 用 `compareTo` 判等；日期时间经 facility `DateUtil` 按 `date-pattern` 渲染）、嵌套 path 用 `.` 连接、IdentityHashMap 环检测（命中返回 CycleDetected）、深度超限返回 DepthExceeded、字段读取失败返回 FieldAccessError。比较器查找顺序：`@CompareWith` > 应用注册的类型级 > 内置 > `Objects.equals` 兜底。
- `render`：`PlainTextRenderer`（三模板可配，占位符 `{label}/{old}/{new}`）、`JsonRenderer`（facility `JsonUtil` 序列化 changes 列表）。
- `autoconfigure`：`ToolboxCompareAutoConfiguration`（`@ConditionalOnProperty(prefix="toolbox.compare", name="enabled", havingValue="true", matchIfMissing=true)`；bean：registry、engine、两个 renderer、`toolboxCompareMessageSource`，全部 `@ConditionalOnMissingBean`）+ `ToolboxCompareProperties`（01 §6 全字段）+ imports 文件 + i18n bundle（默认 + zh_CN）。
- 模块 pom 依赖：compile：`cn.code91:server-facility`、`spring-boot-autoconfigure`；test：`spring-boot-starter-test`、`archunit-junit5`。
- 测试（TDD，先红后绿）：
  - 引擎单测矩阵（01 §9）：各叶子类型（含 BigDecimal `1.0` vs `1.00` 视为相等、枚举、LocalDate/LocalDateTime/OffsetDateTime）、嵌套对象、BY_INDEX 集合（等长改动 + 长度不等）、Map、环、深度超限、nullAsEmpty 两态、`@CompareIgnore`、`@CompareWith` 优先级覆盖、record 与 JavaBean 各一套、messageKey 解析与两级回退；
  - 渲染：PlainText 模板替换断言；JsonRenderer 输出可被反序列化回等价 changes；
  - 装配（ApplicationContextRunner）：enabled=false 全不装配；用户自定义 `DiffEngine` bean 时默认让位；应用注册 `ValueComparator` 被收集进 registry；freeze 后 register 抛异常；
  - ArchUnit 三规则（全局约束 8）。

验收：`mvn -q verify -pl compare-enhancement` 全绿（jacoco 门达标、analyze 零告警、输出无堆栈噪音）。

## Task 3: storage-enhancement P1（三 adapter + 上传守卫 + 装配 + 测试）

必读设计（按序）：docs/design/00-overview.md §2–§4、docs/design/05-storage-enhancement.md 全文。P1 范围 = 05 文档 §10 P1 行，外加以下**控制器裁定**：

- 裁定 A（分片整体推迟）：P1 的 `ObjectStore` 接口**不含**任何 multipart 方法（05 §4.1 的分片族随 P2 一并加入，避免三个 adapter 出现成片 NotSupported 死桩）；`StorageError` P1 只含 6 型：NotFound/AccessDenied/NetworkError/ValidationError/ProviderError/NotSupported（MultipartError 随 P2 加入）。
- 裁定 B（凭证）：aws-s3——properties 提供可选 `access-key-id`/`access-key-secret`，成对给出走 StaticCredentialsProvider（MinIO 场景），否则走 SDK DefaultCredentialsProvider 链；aliyun-oss——`ecs-ram-role` 优先，否则 AK/SK，两者皆缺在 factory 构造期抛 `IllegalStateException`（消息含配置路径）。两个 factory 均在构造期完成凭证/参数校验（fail-fast）。
- 裁定 C（local 预签名）：local adapter 的 `presignedPut/Get` 返回 `Err(NotSupported)`；`PresignedUrl` record 含 url/expiresAt/method。
- 裁定 D（流返回）：`get()` 返回 `Result<InputStream, StorageError>`，调用方负责关闭；P1 不提供 `getAsBytes`。

交付物（基础包 `cn.code91.toolbox.storage`）：

- `core`：`ObjectStore`（bucketName/put/get/list/delete/exists/presignedPut/presignedGet，签名见 05 §4.1 去掉分片族）、`ObjectStoreRegistry` + `DefaultObjectStoreRegistry`（未知逻辑桶抛 `IllegalArgumentException`，消息含 `toolbox.storage.<type>.buckets` 配置引导）、`StorageError` sealed 6 型（record，均含 message，按需带 key/cause/providerCode）、`ObjectMetadata`（key/size/contentType/etag/lastModified）、`ObjectKey`（key/size/lastModified）、`PresignedUrl`。
- `guard`：`UploadCandidate`（filename/declaredContentType/size/peekable 流）、`StorageGuard` seam、`DefaultStorageGuard`——检查顺序：facility `Filenames` 清洗与危险扩展名 → max-size → 配置叠加的 blocked-extensions → `verify-mime=true` 时 facility `MimeTyping` 魔数嗅探与声明 Content-Type 比对；任何违规返回 `Err(ValidationError)` 且消息说明被拦原因。
- `aws`：`AwsS3ObjectStore` + `AwsS3ClientFactory`（endpoint/region/path-style-access/凭证裁定 B）+ `AwsS3ErrorMapper`（NoSuchKeyException→NotFound；403/AccessDenied→AccessDenied；SdkClientException 及 IO 类→NetworkError；其余 S3Exception→ProviderError 带 errorCode）。
- `aliyun`：`AliyunOssObjectStore` + `AliyunOssClientFactory` + `AliyunOssErrorMapper`（NoSuchKey/404→NotFound；InvalidAccessKeyId/403→AccessDenied；ClientException→NetworkError；其余 OSSException→ProviderError）。
- `local`：`LocalFsObjectStore`——root-dir 下按逻辑桶建子目录；key 经 `Filenames` 校验，非法（穿越等）返回 `Err(ValidationError)`；读写走 `PathIo`；list 按前缀；presigned 见裁定 C。
- `autoconfigure`：`ToolboxStorageAutoConfiguration`（结构见 05 §5：空 registry 兜底 + 三个 nested `@Configuration(proxyBeanMethods=false)` 按 `toolbox.storage.type` 互斥，aws/aliyun 嵌套类上加 `@ConditionalOnClass(name=…)`）+ `ToolboxStorageProperties`（05 §6 全字段，buckets 为 `Map<String, BucketProperties>`）+ `StorageGuard` bean + imports 文件。
- 模块 pom：compile：`server-facility`、`spring-boot-autoconfigure`；optional：`software.amazon.awssdk:s3`、`com.aliyun.oss:aliyun-sdk-oss`；test：`spring-boot-starter-test`、`archunit-junit5`、`org.testcontainers:minio`、`org.testcontainers:junit-jupiter`。
- 测试（TDD，先红后绿）：
  - 守卫矩阵：路径穿越文件名、危险扩展名、超 max-size、配置黑名单扩展、MIME 伪装（声明 image/jpeg 实为 zip 魔数）、verify-mime=false 放行；
  - local adapter：全 op 回环、`../../etc/passwd` 型 key 拦截、presigned NotSupported、前缀 list；
  - 两个 ErrorMapper：构造 SDK 异常实例做映射表穷尽单测；
  - 装配（ApplicationContextRunner）：type 三态互斥（各 type 下另两个 adapter bean 不存在）、未配 type→空 registry 且 `get()` 消息含配置引导、`FilteredClassLoader` 模拟缺 SDK 类→对应 nested 不装配、用户覆盖 `ObjectStoreRegistry` 让位；
  - 集成（`AwsS3ObjectStoreIT`）：Testcontainers `MinIOContainer`，全 op 回环 + presignedPut/Get 用 `java.net.http.HttpClient` 真实 PUT/GET 断言内容一致；标注 `@Testcontainers(disabledWithoutDocker=true)`；
  - ArchUnit 三规则（core/spi/guard 不得引用 `software.amazon.**` 或 `com.aliyun.**` 类型）。
  - aliyun adapter 不做真云 IT（真云冒烟进 USAGE 清单，不进 CI）。

验收：`mvn -q verify -pl storage-enhancement` 全绿（含 MinIO IT；jacoco 门达标、analyze 零告警）。
