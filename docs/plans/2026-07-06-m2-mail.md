# M2 实施计划（mail-enhancement P1）

> 依据：[00-overview](../design/00-overview.md)、[04-mail](../design/04-mail-enhancement.md)。
> 执行方式：subagent-driven development（同 M0+M1）。前序：M0+M1 已合入 master（ad9a5c7），父 POM 质量门与 greenmail-junit5 2.1.3 版本 pin 已就绪。

## Global Constraints（全局约束，违反即缺陷）

1. Java 21、Spring Boot 3.5.10；版本一律由父 pom dependencyManagement 管理，子模块不写版本号。
2. 基础包 `cn.code91.toolbox.mail`；配置前缀 `toolbox.mail.*`；唯一装配类 `ToolboxMailAutoConfiguration`，经 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 注册。
3. 对外 API 返回 `cn.code91.facility.result.Result<T, MailError>`；`MailError` 为 sealed interface + record 实现；jakarta.mail/Spring Mail 的异常一律在 smtp adapter 内收敛为错误值，不得从公开 API 外抛。
4. 可选依赖必须 `<optional>true</optional>`，装配处用 `@ConditionalOnClass(name = "全限定名字符串")` 探测。
5. 每个默认 bean 挂 `@ConditionalOnMissingBean`（Seam）。
6. 复用 facility、禁止重造：`Filenames`/`MimeTyping`（附件守卫）、`RateLimiterUtil`（限速）、`LogUtil`（日志，收件人输出走脱敏）、`Result`。注意实测教训：`MimeTyping` 静态硬依赖 optional 的 tika-core，使用前必须 `Class.forName` 探测（与 storage-enhancement 的守卫同一模式，语义保持一致）。
7. 不使用 Lombok；值对象用 record；注释与 Javadoc 用中文，只写代码本身表达不了的约束。
8. 质量门：jacoco BUNDLE INSTRUCTION/LINE ≥ 0.80、BRANCH ≥ 0.70；ArchUnit 三规则（包无环；autoconfigure 不被主包依赖；core/spi/template 不得引用 jakarta.mail.** 或 org.springframework.mail.**——Seam 与值对象必须在缺 starter-mail 时可加载）；`mvn verify` 全绿且测试输出无告警噪音。
9. TDD：先写失败测试再实现，报告含 RED/GREEN 证据。
10. git 提交：中文一行主题；结尾带 `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`。
11. 模块 i18n：P1 不设 MessageSource/bundle（错误消息为面向开发者的中文硬编码，与 storage 一致）；若未来引入，键一律 `toolbox.mail.*` 命名空间且示例键只进 src/test/resources（教训：compare 终审 I4，防 facility 聚合池污染）。

## Task 1: mail-enhancement P1（多账号派发 + 模板 + 沙箱 + 装配 + 测试）

必读设计（按序）：docs/design/00-overview.md §2–§4、docs/design/04-mail-enhancement.md 全文。P1 范围 = 04 文档 §10 P1 行，外加以下**控制器裁定**（本 brief 优先于设计文档的含糊处）：

- 裁定 A（P1 范围收窄）：P1 = 多账号 SMTP + 文本/HTML/附件 + 轻量 `${var}` 模板 + 沙箱 + 重试/限速 + `MailSendListener` 回调 + `adopt-spring-mail` 收编。以下明确**排除**（P2/P3，不得出现相关代码）：Thymeleaf adapter、模板多语言回退（`MailTemplateRenderer` 接口签名保留 `Locale` 参数以稳定 API，但 P1 默认实现忽略之并 Javadoc 注明，模板仅查 `classpath:mail-templates/{name}.html`）、inline cid 资源、HTTP API 厂商、账号故障转移、发送队列持久化。
- 裁定 B（沙箱 fail-fast）：`sandbox.enabled=true` 且 `redirect-to` 与 `allowed-domains`/`allowed-recipients` 均未配置 → 启动期抛 `IllegalStateException`（消息含配置路径与两种配置方式），防"以为拦了其实全放行"；沙箱启用时装配期打一条显眼 WARN（经 LogUtil）。决策顺序按 04 §4.3：redirect-to 优先，其次白名单，命中放行、未命中 `Err(SandboxBlocked)` 携带原收件人；redirect 时原收件人写入邮件头 `X-Original-To` 并记日志。
- 裁定 C（重试与限速语义）：重试仅针对 `ConnectError`（连接/握手类），指数退避（`retry.max-attempts` 默认 3、`retry.backoff` 默认 2s，倍增），同步阻塞式；`AuthError`/`InvalidRecipient` 等直返不重试。限速为账号级，`rate-limit-per-minute`（0=不限）经 facility `RateLimiterUtil` 换算实现（换算方式实现者定，Javadoc 写明突发语义），命中返回 `Err(RateLimited)` 含建议等待时长，**不阻塞等待**。
- 裁定 D（账号解析与收编）：`adopt-spring-mail=true`（默认）且容器存在消费方自己的 `JavaMailSender` bean 时，将其收编为名为 `spring` 的账号（该账号无 from 配置时发送方从 MailMessage 显式取，缺失即 `Err(ConfigError)`）。primary 解析顺序：显式 `toolbox.mail.primary` > 唯一账号 > `spring`。零账号（含无 spring bean）→ 空 registry 兜底，`get()`/`primary()` 抛带配置引导的 `IllegalArgumentException`（与 storage 空 registry 同模式）。
- 裁定 E（附件守卫镜像 storage 语义）：`AttachmentGuard` 默认实现检查顺序 = `Filenames` 清洗与危险扩展名 → `attachment.max-size` → 配置叠加 `attachment.blocked-extensions` → `attachment.verify-mime=true` 时 MIME 魔数嗅探与声明比对（tika `Class.forName` 探测，缺失 fail-closed 带依赖引导；`verify-mime` 默认 **false**）。附件为内存 `byte[]`（P1 不做流式），违规返回 `Err(AttachmentRejected)` 含被拦原因。
- 裁定 F（异步）：`sendAsync` 返回 `CompletableFuture<Result<MailReceipt, MailError>>`，执行器优先复用容器中 facility 的 TaskExecutor（虚拟线程），不可得时模块内共享一个虚拟线程执行器；不得每次发送新建执行器。

交付物（基础包 `cn.code91.toolbox.mail`）：

- `core`：`MailDispatcher`（send/sendAsync）、`MailAccountRegistry`（get(name)/primary()）、`MailMessage`（不可变 + builder：to/cc/bcc/replyTo/subject/text/html(templateName,vars)/rawHtml/attach，**不得 import jakarta.mail 类型**）、`Attachment`（filename/contentType/bytes record）、`MailReceipt`（messageId/account/sentAt）、`MailError` sealed 9 型（ConfigError/AuthError/ConnectError/InvalidRecipient/TemplateError/AttachmentRejected/RateLimited/SandboxBlocked/ProviderError，record，均含 message，按需带原因/建议等待/原收件人）。
- `spi`：`MailTemplateRenderer`（name() + render(templateName, vars, locale) → Result<String, MailError>）、`MailSendListener`（onSuccess/onFailure default 方法；ObjectProvider 收集，全部回调，回调异常吞掉并记日志不影响主流程）、`AttachmentGuard`（check(Attachment) → Result<Void, MailError>）。
- `template`：`SimpleTemplateRenderer`——`classpath:mail-templates/{name}.html` 加载 + `${var}` 占位替换（缺模板→TemplateError 含路径；缺变量→保留原样或空串，实现者定并测试钉住）。
- `guard`：`DefaultAttachmentGuard`（裁定 E）。
- `sandbox`：`SandboxPolicy`（裁定 B 的决策逻辑，独立可测）。
- `smtp`：`JavaMailSenderFactory`（按账号配置构造 `JavaMailSenderImpl`：host/port/ssl/username/password/protocol/timeout，构造期校验必填项 fail-fast）、`SmtpMailDispatcher`（组装：限速 → 沙箱 → 模板渲染 → 附件守卫 → MimeMessage 构造发送 → 重试 → 回执/回调）、`SmtpErrorMapper`（jakarta.mail 异常收敛：`AuthenticationFailedException`→AuthError；连接/超时类（含 `MailSendException` 内嵌套的连接异常）→ConnectError；`SendFailedException` 无效地址→InvalidRecipient；其余→ProviderError）。
- `autoconfigure`：`ToolboxMailAutoConfiguration`（`@AutoConfiguration(after = MailSenderAutoConfiguration.class)` + `@ConditionalOnClass(name = "jakarta.mail.internet.MimeMessage")` + enabled 开关 matchIfMissing=true；beans：renderer/guard/registry 全部 `@ConditionalOnMissingBean`；registry 构造注入 `ObjectProvider<JavaMailSender>`（裁定 D）与 `ObjectProvider<MailSendListener>`）+ `ToolboxMailProperties`（04 §6 全字段：primary/adopt-spring-mail/accounts map{host,port,ssl,username,password,from,display-name,rate-limit-per-minute,retry{max-attempts,backoff}}/attachment{max-size,blocked-extensions,verify-mime}/sandbox{enabled,redirect-to,allowed-domains,allowed-recipients}）+ imports 文件。
- 模块 pom：compile：`server-facility`、`spring-boot-autoconfigure`；optional：`spring-boot-starter-mail`；test：`spring-boot-starter-test`、`archunit-junit5`、`greenmail-junit5`（版本已 pin 2.1.3，若无法解析可换同 artifact 邻近补丁版并在报告显著披露）。
- 测试（TDD，先红后绿）：
  - **GreenMail 集成**（`*Test` 即可，GreenMail 是内嵌库无需 Docker）：单账号发文本/HTML/附件全回环（断言收到的 MimeMessage 主题/正文/附件名与内容）；多账号并存各自发送；认证失败→AuthError；连接拒绝（未监听端口）→ConnectError 且重试次数可断言（如经 listener 计数或时间窗）；模板渲染端到端；
  - 沙箱矩阵：redirect（断言实际收件人被替换 + `X-Original-To` 头 + 原收件人在 Err 之外正常回执）/白名单命中/未命中→SandboxBlocked/未配置两者→启动 fail-fast/enabled=false 直通，另加"生产误开沙箱但配了 redirect"路径正常；
  - 附件守卫矩阵：危险扩展名/超大小/黑名单/verify-mime 开且 tika 缺失 fail-closed（FilteredClassLoader）/verify-mime=false 放行伪装 MIME；
  - `SmtpErrorMapper` 映射表穷尽单测（构造真实 jakarta.mail 异常实例）；
  - 装配矩阵（ApplicationContextRunner）：缺 jakarta.mail 类全不装配（FilteredClassLoader）；enabled=false 全关；零账号→空 registry 且 get() 消息含配置引导；adopt-spring-mail 收编（含 primary 三级解析各态）；用户覆盖 renderer/guard/registry 让位；listener 多实例全回调且回调抛异常不影响发送结果；
  - `SimpleTemplateRenderer` 单测：正常替换/缺模板/缺变量语义；
  - ArchUnit 三规则（约束 8）。

验收：`mvn -q verify -pl mail-enhancement` 全绿（jacoco 门达标、dependency:analyze 零告警、输出无告警噪音）；随后仓库根 `mvn -q verify` 全绿（不回归 compare/storage）。
