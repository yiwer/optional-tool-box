# M5 实施计划（docs-enhancement P1）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 落地第六模块 docs-enhancement P1——springdoc-openapi 薄封装：配置驱动分组（属性翻译）、Info/安全方案/服务器配置、契约感知三件事（`Result` 泄漏检测、`BaseResponse` schema 命名修饰、通用错误形状文档）、环境门禁（请求期过滤）、JSON/YAML/Postman 导出。

**Architecture:** 依据 [06-docs-enhancement 设计](../design/06-docs-enhancement.md)（含三处"写实现计划前修正"技术澄清，实现以设计文档当前版为准）。单一 `ToolboxDocsAutoConfiguration` 装配入口 + 一个 `EnvironmentPostProcessor`（spring.factories 注册，分组属性翻译）；`core` 包零 springdoc 依赖，springdoc/swagger 类型只出现在 `schema`/`export`/`exposure`/`grouping`/`autoconfigure`。

**Tech Stack:** Java 21、Spring Boot 3.5.10（BOM 管版本）、springdoc-openapi 2.8.x（optional）、JUnit 5 + AssertJ、ApplicationContextRunner + FilteredClassLoader、`@SpringBootTest`(RANDOM_PORT) 内嵌容器 IT（本仓库首个）、Maven（无 mvnw，直接 `mvn`）。

## Global Constraints（全局约束，违反即缺陷）

1. Java 21、Spring Boot 3.5.10；版本由父 pom dependencyManagement 管理，子模块不写版本号。
2. 基础包 `cn.code91.toolbox.docs`；配置前缀 `toolbox.docs.*`；唯一装配类 `ToolboxDocsAutoConfiguration` 经 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 注册。**另有唯一例外注册文件**：`DocsGroupsEnvironmentPostProcessor` 经 `META-INF/spring.factories`（key `org.springframework.boot.env.EnvironmentPostProcessor`）注册——EPP 不是装配类，Boot 3 对 EPP 仍只认 spring.factories，此为机制必需而非破例，需在该文件头注释与模块 README 披露。
3. 对外 API 返 `cn.code91.facility.result.Result<T, DocsError>`；`DocsError` sealed **单变体** `ExportFailed(String message)`（设计 §4.2 收窄说明——不得加回 `InvalidGroupConfig`/`SchemaResolutionFailed` 死桩）。配置期校验错误（分组选择器全空）抛 `IllegalStateException` fail-fast，消息含配置路径（同 storage 缺 `root-dir` 先例）。
4. springdoc 相关依赖全部 `<optional>true</optional>`；装配处 `@ConditionalOnClass(name = "org.springdoc.core.models.GroupedOpenApi")` 字符串探测；`springdoc-openapi-starter-webmvc-ui` 模块 pom **不声明**（消费方按需引入，版本由父 pom 收口）。
5. 每个默认 bean 挂 `@ConditionalOnMissingBean`（Seam）：`OpenAPI`/`DocsExposureGate`/`DocsExporter`/`ResultLeakDetector`。
6. 复用 facility、禁止重造：`Result`、`LogUtil`（告警/操作日志）、`BaseResponse`/`PageBaseResponse`（契约感知的识别目标）、`JsonUtil` 或 springdoc 自带 Jackson（Postman 转换，不自建序列化）。**不自设 SPI**：定制扩展点 = springdoc 原生 `OpenApiCustomizer`/`OperationCustomizer`/`ModelConverter` bean 收集机制（设计 §4.4 修正）。
7. 不使用 Lombok；值对象用 record；注释与 Javadoc 用中文，只写代码本身表达不了的约束。
8. 质量门：jacoco BUNDLE INSTRUCTION/LINE ≥ 0.80、BRANCH ≥ 0.70；ArchUnit 三规则（顶层切片包无环；`autoconfigure` 不被其它包依赖；**`core` 不依赖 `org.springdoc..`/`io.swagger..` 类型**——缺 springdoc 时门面可加载）；`mvn verify` 全绿且输出无告警噪音；`maven-dependency-plugin` analyze failOnWarning（直接使用必须显式声明，误报按 storage/compare 先例加注释抑制）。

#### 控制器裁定（实现者不得自行改判，有疑问回报控制器）

- **裁定 A（P1 范围）**：P1 = 分组属性翻译 + `OpenAPI` bean（Info/servers/security-schemes）+ 契约感知三件事 + 环境门禁过滤器 + 三格式导出 + 装配 + i18n 基础设施。**明确排除**（不得出现相关代码/死桩/预留分支）：`DocsAccessGuard`、knife4j、版本快照 diff、WebFlux、泄漏检测 `strict` 模式、按 endpoint 枚举具体业务错误。
- **裁定 B（分组翻译）**：`DocsGroupsEnvironmentPostProcessor` 用 `Binder.get(environment)` 绑定 `toolbox.docs.groups`（Map<String, Group>，Group = `packages-to-scan`/`paths-to-match`/`paths-to-exclude` 三个 `List<String>`）；`toolbox.docs.enabled=false` 时直接返回；每组三选择器全空 → `IllegalStateException`（消息含 `toolbox.docs.groups.<name>` 路径与"至少配置 packages-to-scan 或 paths-to-match 之一"引导）；产出 `MapPropertySource("toolboxDocsGroups")`：`springdoc.group-configs[i].group=<name>` + 各选择器逐项 `[j]` 索引展开；**addLast**（消费方显式写的 springdoc 原生属性优先）；无 groups 配置时不添加 property source。组名即 map 键，天然唯一，无重名校验。
- **裁定 C（契约感知三件事）**：
  1. `ResultLeakDetector` implements `SmartInitializingSingleton`：`afterSingletonsInstantiated()` 里经构造注入的 `ObjectProvider<RequestMappingHandlerMapping>` 遍历 `getHandlerMethods()`；判定"声明返回类型 raw class == `cn.code91.facility.result.Result`，或 `ResponseEntity<X>` 时取 X 再判"；命中 → `LogUtil` warn 每方法一条，含 `控制器类简名#方法名` 与"应经 BaseResponse.fromResult(...) 转换后再暴露"引导；**不抛异常不阻断启动**；受 `toolbox.docs.contract-awareness.result-leak-detection`（默认 true）开关控制（false 时 bean 不注册，`@ConditionalOnProperty(matchIfMissing = true)`）。
  2. schema 命名修饰（两个 `ModelConverter`）：`BaseResponse<X>` → schema 名 `{X 的 swagger 默认命名片段}Response`（如 `UserDtoResponse`、`ListUserDtoResponse`）；`PageBaseResponse<X>` → `{X}PageResponse`；raw/未参数化类型不动；解析失败或命名冲突 → 退回默认命名 + `LogUtil` warn（**不产生错误值、不抛异常**——设计 §4.2 收窄的前提）。字段结构不改（springdoc 本就解析正确，见设计 §1 技术澄清）。
  3. 错误形状：`ErrorShapeOpenApiCustomizer`（springdoc `GlobalOpenApiCustomizer`）向 `components.schemas` 注册 `ToolboxErrorResponse`（code: integer int32 / message: string / description: string / data: 可空 object）；`ErrorShapeOperationCustomizer`（springdoc `OperationCustomizer`）对**尚无 `default` 响应**的 operation 追加 `default` → `$ref: #/components/schemas/ToolboxErrorResponse`，description "统一错误响应（BaseResponse 错误分支通用形状）"；已有 `default` 的 operation 不动（尊重应用显式注解）。
- **裁定 D（环境门禁）**：`DefaultDocsExposureGate(ToolboxDocsProperties)`：激活 profile 与 `exposure.production-profiles`（默认 `[prod, production]`）有交集 **且** `exposure.expose-in-production=false`（默认）→ `isExposed` 返回 false；其余一律 true。`DocsExposureFilter` extends `OncePerRequestFilter`：`isExposed()` 为 false → `response.sendError(404)` 并返回；经 `FilterRegistrationBean` 注册，urlPatterns 计算：`{apiDocsPath, apiDocsPath + "/*", swaggerUiPath, "/swagger-ui/*", "/toolbox/docs/export"}`，其中 `apiDocsPath` 读 `springdoc.api-docs.path`（Environment，默认 `/v3/api-docs`）、`swaggerUiPath` 读 `springdoc.swagger-ui.path`（默认 `/swagger-ui.html`）——**读属性不硬编码**（设计风险 2）。gate 判定在 doFilter 时求值（非注册期缓存），保证测试与运行语义一致。
- **裁定 E（导出）**：`SpringdocDocsExporter` 构造注入 `ObjectProvider<OpenApiWebMvcResource>`（默认文档）与 `ObjectProvider<MultipleOpenApiWebMvcResource>`（分组文档）+ `ToolboxDocsProperties`（组名合法性校验的事实来源）：group 非空且不在 `toolbox.docs.groups` 键集 → `Err(ExportFailed)` 含已配置组列表；JSON/YAML 分别调 springdoc 资源 bean 的 `openapiJson`/`openapiYaml` 公开方法（`HttpServletRequest` 经 `RequestContextHolder` 获取，模块本就 Servlet-only）；POSTMAN = `PostmanCollectionWriter`：以 JSON spec 为输入（Jackson tree），产出 Postman collection v2.1——`info.name` 取 spec `info.title`（空则 "API"）、`info.schema` 固定 v2.1.0 URL、每 `paths.<path>.<method>` 一个 item：`name` 取 operation `summary`→`operationId`→`"METHOD path"` 三级兜底、`request.method` 大写、`request.url.raw` = `{{baseUrl}}` + path、`request.url.host=["{{baseUrl}}"]`、`request.url.path` 为分段数组；不含 auth/body/示例（P1 最小可用，README 披露）。springdoc 内部抛错（如 spec 生成失败）→ 捕获收敛为 `Err(ExportFailed)`，不外抛。`DocsExportController`（`@RestController`，`GET /toolbox/docs/export`）：`Ok` → 200，Content-Type 按格式（json→`application/json`、yaml→`application/yaml`、postman→`application/json`）+ `Content-Disposition: attachment; filename=...`；`Err(ExportFailed e)` → 404 + `BaseResponse.err(404, e.message())`；`format` 参数非法值 → 400 + `BaseResponse.err(400, ...)`（Spring 枚举绑定失败的默认行为可接受，但需测试钉住实际形态）。`toolbox.docs.export.enabled=false` → controller 与 exporter bean 均不注册（`@ConditionalOnProperty(matchIfMissing = true)`）。
- **裁定 F（版本 pin 与依赖账目）**：父 pom properties 加 `<springdoc.version>2.8.9</springdoc.version>`（**落地第一步实测复核 Maven Central 上与 Boot 3.5.10 兼容的最新 2.8.x GA，若更高则取用并在提交信息披露**）；dependencyManagement pin `org.springdoc:springdoc-openapi-starter-common`、`springdoc-openapi-starter-webmvc-api`、`springdoc-openapi-starter-webmvc-ui`（ui 仅为消费方收口版本，模块不依赖）。模块 pom：compile = `server-facility`、`spring-boot-autoconfigure` + 字节码直接引用的 spring 薄层（`spring-boot`/`spring-context`/`spring-core`/`spring-web`/`spring-webmvc`，以 analyze 实际报告为准逐条声明并注释因果，同 storage 先例）+ `jakarta.servlet:jakarta.servlet-api`（**provided**——Filter API 编译需要，运行时由消费方容器提供）；optional = `springdoc-openapi-starter-common`、`springdoc-openapi-starter-webmvc-api` + 若字节码直接引用 swagger 类型（`io.swagger.v3.oas.models.*`/`io.swagger.v3.core.converter.*`）则显式声明 `io.swagger.core.v3:swagger-models-jakarta`/`swagger-core-jakarta`（版本在父 pom pin，对齐 springdoc 传递版本）；test = `spring-boot-starter-test`、`spring-boot-test`、`junit-jupiter-api`、`assertj-core`、`archunit-junit5`、`archunit`、`spring-boot-starter-web`（**test scope**——IT 内嵌 Tomcat）。optional 依赖在模块自身 test classpath 可见（optional 只影响传递性），IT 无需重复声明 springdoc。
