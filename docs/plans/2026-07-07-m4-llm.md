# M4 实施计划（llm-enhancement P1）

> 依据：[00-overview](../design/00-overview.md)、[03-llm](../design/03-llm-enhancement.md)。
> 执行方式：subagent-driven development。前序：M0+M1+M2+M3 已在 master（8003f7d）。llm 是最后一个模块。
> artifactId 已在 M0 修正为 `llm-enhancement`（无需再改）。无需 Docker（HTTP 契约测试用 WireMock，非容器）。

## Global Constraints（全局约束，违反即缺陷）

1. Java 21、Spring Boot 3.5.10；版本由父 pom dependencyManagement 管理，子模块不写版本号。
2. 基础包 `cn.code91.toolbox.llm`；配置前缀 `toolbox.llm.*`；唯一装配类 `ToolboxLlmAutoConfiguration`，经 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 注册。
3. 对外 API 返回 `cn.code91.facility.result.Result<T, LlmError>`；`LlmError` 为 sealed interface + record 实现；HTTP/解析异常一律在 openai adapter 内收敛为错误值，不得从公开 API 外抛。
4. 可选依赖必须 `<optional>true</optional>`，装配处用 `@ConditionalOnClass(name = "全限定名字符串")` 探测（P1 无重可选依赖——Spring AI 桥接是 P3）。
5. 每个默认 bean 挂 `@ConditionalOnMissingBean`（Seam）。
6. 复用 facility、禁止重造：`HttpClients`/RestClient（同步调用通道，**不自建 HTTP 客户端**）、`MaskUtil`（提示词/响应日志脱敏）、`RateLimiterUtil`（客户端限流，签名 `acquire(key,permits,capacity,permitsPerSecond)` → `RateLimitResult(allowed,remaining,retryAfterMillis)`，无 bean 降级放行）、`JsonUtil`（结构化输出解析，CANONICAL 命名空间）、`LogUtil`（写前自动脱敏）、`Result`。
7. 不使用 Lombok；值对象用 record；注释与 Javadoc 用中文，只写代码本身表达不了的约束。
8. 质量门：jacoco BUNDLE INSTRUCTION/LINE ≥ 0.80、BRANCH ≥ 0.70；ArchUnit 三规则（包无环；autoconfigure 不被主包依赖；core/spi 门面不依赖 openai adapter 具体类与 HTTP 客户端类型——保证门面可独立加载）；`mvn verify` 全绿且测试输出无告警噪音。
9. TDD：先写失败测试再实现，报告含 RED/GREEN 证据。OpenAiErrorMapper 用真实 HTTP 状态/错误体构造断言。
10. git 提交：中文一行主题；结尾带 `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`。
11. i18n：P1 无 MessageSource/bundle（与 storage/mail/database 一致；错误为面向开发者中文）。若未来引入，键 `toolbox.llm.*` 前缀、示例键只进 src/test/resources（compare 终审 I4 教训）。
12. **敏感信息**：api-key 绝不出现在 toString/日志/错误消息；提示词/响应日志经 MaskUtil 脱敏（默认开）；配置示例强制 `${ENV_VAR}` 占位符。

## Task 10: llm-enhancement P1（多模型注册 + OpenAI 兼容适配 + 结构化输出 + 装配 + 测试）

必读（按序）：docs/design/00-overview.md §2–§4、docs/design/03-llm-enhancement.md 全文、上文 Global Constraints。**同仓先例**：compare/storage/mail/database 已实现并终审——注册表空兜底与引导消息、装饰链编排（看 mail 的 SmtpMailDispatcher：限速→…→发送→重试→回调）、装配矩阵（ApplicationContextRunner + FilteredClassLoader）、ArchUnit 三规则、record Properties、MaskUtil 脱敏落法，直接复用其模式。P1 范围 = 03 文档 §10 P1 行，外加以下**控制器裁定**：

- 裁定 A（P1 范围与接口边界）：P1 = 同步 `chat` + `chatStructured` + 错误映射 + 重试 + 客户端限流 + 脱敏日志 + 用量回调 + classpath 提示词模板 + 多模型注册表（primary）。**明确排除**（P2/P3，不得出现相关代码）：SSE 流式 `chatStream`、`chatAsync`、Spring AI 桥接、Anthropic 原生 type、embedding/向量、function calling。**P1 的 `LlmClient` 接口只含 `chat` 与 `chatStructured` 两个方法**——不要放 `chatStream` 死桩（它需要 SSE 基建，P2 再加；这与 mail 保留 Locale 参数不同，chatStream 是整个方法，留桩即污染）。
- 裁定 B（结构化输出鲁棒性）：`chatStructured(request, Class<T>)` 必须在解析前**剥壳**——去除 markdown 围栏（```json … ```）与前后散文，再经 `JsonUtil` 反序列化为目标类型；解析或校验失败返回 `Err(SchemaMismatch)` 且**携带模型原始文本**（便于排查）。不得依赖厂商"JSON mode"开关（并非所有兼容端点支持）；P1 策略 = 提示词指示 JSON + 鲁棒解析。可选择在配置支持时附带 `response_format`，但解析必须无论如何鲁棒。
- 裁定 C（重试语义）：仅对 `RateLimited`（若响应含 Retry-After 则尊重）/`Timeout`/`NetworkError` 重试，指数退避（`max-retries` 默认 2、退避基数可配，倍增），同步阻塞式；`AuthError`/`ContentFiltered`/`ContextLengthExceeded`/`SchemaMismatch`/`ProviderError` 直返不重试（内容过滤重试无意义且烧钱）。
- 裁定 D（注册表与 primary）：`LlmClientRegistry.get(modelName)` / `primary()`；配置为 models map，每条含 `type: openai-compatible` + base-url + api-key + model + 可选 options（temperature/max-tokens/timeout/max-retries/rate-limit-qps）。primary 解析：显式 `toolbox.llm.primary` > 唯一模型 > 报错。零模型 → 空 registry 兜底，`get()`/`primary()` 抛带配置引导的 `IllegalArgumentException`（同 storage/mail 模式）。启动 fail-fast：模型条目缺 base-url/api-key → `IllegalStateException` 指明配置路径。
- 裁定 E（同步传输）：同步 POST `/chat/completions` 走 facility `HttpClients`/RestClient（**不用 JDK HttpClient**——那是 P2 SSE 的）。若必须直接用 Spring `RestClient` 类型，模块 pom 显式声明 spring-web（compile）并披露；优先用 facility 门面避免直接依赖。
- 裁定 F（限流换算）：`rate-limit-qps`（每秒许可，0=关闭）直接喂 `RateLimiterUtil.acquire`（capacity 语义与突发容忍实现者定并 Javadoc 写明）；命中返回 `Err(RateLimited)` 含建议等待（取自 RateLimitResult.retryAfterMillis），**不阻塞等待**。key 按模型名隔离。

交付物（基础包 `cn.code91.toolbox.llm`）：

- `core`：`LlmClient`（**仅 chat + chatStructured**）、`LlmClientRegistry`（get/primary）、`ChatRequest`（messages + options，builder；含 `user(String)` 便捷构造）、`Message`（role + content，role 枚举 SYSTEM/USER/ASSISTANT）、`ChatOptions`（temperature/maxTokens/timeout… 全可选 record）、`ChatResponse`（content/usage/finishReason/rawModel）、`Usage`（promptTokens/completionTokens/totalTokens）、`LlmError` sealed 8 型（AuthError/RateLimited(retryAfter)/ContextLengthExceeded/ContentFiltered/SchemaMismatch(rawText)/Timeout/NetworkError/ProviderError(providerCode)，record，均含 message）。
- `spi`：`PromptTemplateLoader`（load(name) → Optional<String>；内置 classpath: prompts/*.md）、`UsageListener`（onUsage(modelName, usage, latency, traceId)；ObjectProvider 收集，全部回调，回调异常吞掉记日志不影响主流程）。
- `prompt`：`PromptTemplates.render(name, vars)` → `Result<String, LlmError>`（`${var}` 占位；缺模板→ TemplateError 或复用 LlmError 中语义合适型并 Javadoc 说明；缺变量语义实现者定并测试钉住）。
- `openai`：`OpenAiCompatibleClient`（实现 LlmClient；同步走 RestClient；织入 限流→脱敏日志→请求→重试→用量回调 装饰链）+ `OpenAiErrorMapper`（HTTP 401/403→AuthError；429→RateLimited 带 Retry-After；400 上下文超长关键字→ContextLengthExceeded；内容安全→ContentFiltered；超时→Timeout；连接/IO→NetworkError；其余→ProviderError 带 providerCode）+ 请求/响应 DTO（内部 record，不外泄）。
- `autoconfigure`：`ToolboxLlmAutoConfiguration`（enabled matchIfMissing=true；`LlmClientRegistry` bean `@ConditionalOnMissingBean`，构造注入 `ObjectProvider<UsageListener>`，遍历 models 按 type 构造 adapter 并织入重试/限流/脱敏/用量装饰）+ `PromptTemplateLoader` bean（classpath 默认）+ `ToolboxLlmProperties`（03 §6 全字段：enabled/primary/log{enabled,mask-content}/models map{type,base-url,api-key,model,temperature,timeout,max-retries,rate-limit-qps}）+ imports 文件。
- 模块 pom：compile：`server-facility`、`spring-boot-autoconfigure`（若直接用 RestClient 类型则加 spring-web，披露）；test：`spring-boot-starter-test`、`archunit-junit5`、`org.wiremock:wiremock`（**父 pom 需 pin wiremock 版本——按 M2 授权 b：直接使用的 test 构件缺版本管理，补 pin 并披露；版本取最新 3.x GA**）。
- 测试（TDD，先红后绿）：
  - **WireMock 契约测试**（无 Docker）：模拟 OpenAI 兼容 `/chat/completions`——正常响应解析（content/usage/finishReason）；401→AuthError；403→AuthError；429+Retry-After→RateLimited 且重试次数可断言（尊重 Retry-After）；超时→Timeout+重试；连接错误→NetworkError+重试；400 上下文超长→ContextLengthExceeded 不重试；内容安全响应→ContentFiltered 不重试；错误体各形态 → OpenAiErrorMapper 映射表穷尽；
  - `chatStructured` 矩阵：干净 JSON→目标类型；markdown 围栏包裹的 JSON（现实常见，需剥壳）→成功；前后带散文的 JSON→剥壳成功；非法 JSON→SchemaMismatch 且携带原始文本；
  - 装配矩阵（ApplicationContextRunner）：无 models 配置→空 registry 可启动且 get() 消息含配置引导；缺 api-key→启动失败且报错含配置路径；primary 三级解析各态；UsageListener 多实例全回调且回调抛异常不影响 chat 结果；用户覆盖 registry/client 让位；
  - 脱敏验证：含手机号/身份证的 prompt 落日志后必须已脱敏（断言日志输出经 MaskUtil）；api-key 不出现在任何日志/错误消息（断言）；
  - `PromptTemplates` 单测：正常替换/缺模板/缺变量语义；
  - ArchUnit 三规则（约束 8，core/spi 不得引用 openai 包具体类与 HTTP 客户端类型）。

验收：`mvn -q verify -pl llm-enhancement` 全绿（jacoco 门达标、analyze 零告警、输出无噪音）；随后仓库根 `mvn -q verify` 全绿（不回归 compare/storage/mail/database）。
